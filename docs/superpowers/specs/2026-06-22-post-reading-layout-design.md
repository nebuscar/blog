# 文章阅读布局设计

## 目标

将文章详情页改为稳定的三栏阅读布局：

- 左侧：树形目录，可展开、收起并拖动调节宽度。
- 中间：文章正文。
- 右侧：关联文档图谱。
- 浏览器左右边缘：随时可用的上一篇、下一篇按钮。

## 桌面布局

页面在宽度不小于 `1280px` 时启用三栏：

- 左侧目录默认宽度 `280px`，允许在 `220px` 至 `420px` 范围内拖动。
- 中间正文保持 `48rem` 的最大阅读宽度。
- 右侧图谱默认宽度 `280px`，保持固定，不参与拖动。
- 左侧目录与右侧图谱分别使用 sticky 定位，并限制在视口高度内滚动。

左侧目录和正文之间提供拖动条。拖动宽度写入
`localStorage`，双击拖动条恢复默认值。

## 树形目录

目录读取 Astro 提供的 `H2`、`H3`、`H4` 标题：

- `H2` 为一级树节点。
- `H3` 为 `H2` 的子节点。
- `H4` 为 `H3` 的子节点。
- 有子节点的条目显示展开或收起按钮。
- 当前阅读章节及其父级路径自动展开。
- 当前章节高亮并自动滚动到目录可视区域。
- 标题允许换行，不通过缩小字号强行塞进一行。
- 顶部提供“全部展开 / 全部收起”控制。

## 翻页交互

桌面端上一篇、下一篇按钮贴近浏览器左右边缘垂直居中，不根据正文、
目录或图谱宽度计算位置：

- 默认只显示圆形方向按钮。
- 悬停或键盘聚焦时向页面内部展开文章标题。
- 按钮与页面边缘保留安全距离。
- 支持 `Alt + ←` 和 `Alt + →` 快捷键。

在小于 `1280px` 的设备上隐藏边缘按钮，使用右下角可展开的文章导航
控件，避免遮挡正文。文章末尾仍保留静态上一篇、下一篇卡片作为无脚本
和无障碍兜底。

## 响应式布局

- `>= 1280px`：左目录、中正文、右图谱三栏。
- `768px–1279px`：正文单栏，目录和图谱以折叠卡片放在正文上方。
- `< 768px`：正文单栏，目录和图谱默认折叠；使用移动端文章导航控件。

## 验证

- 单元测试验证标题层级构建、展开路径和侧栏宽度限制。
- Astro 类型检查及完整生产构建。
- 桌面宽度验证三栏不重叠，拖动后翻页按钮位置不变。
- 移动端验证目录、图谱和文章导航不遮挡正文。

# 探索型生信分析项目

## 1 适用范围

适用于：

- 单细胞 / 单核 RNA-seq 分析
- bulk RNA-seq 分析
- DEG、GSEA、WGCNA、GSVA
- 公共数据库挖掘
- 论文复现
- 硕士课题中的探索性分析

核心逻辑：

> 数据输入 → 分析步骤 → 中间对象 → 图表结果 → 方法记录

## 2 推荐顶层结构

```text
ProjectName/
├── README.md
├── code/
├── utils/
├── data/
├── metadata/
├── results/
└── docs/
```

## 3 目录职责

| 目录 | 作用 |
|---|---|
| `README.md` | 项目入口说明、数据来源、运行顺序 |
| `code/` | 主分析流程脚本，一步一个脚本 |
| `utils/` | 可复用函数、marker、调色板、模板 |
| `data/raw/` | 原始数据，不手动修改 |
| `data/processed/` | 处理后的 RDS、h5ad、矩阵、中间对象 |
| `metadata/` | 正式输入样本信息、分组表、字段说明 |
| `results/figs/` | 分析图 |
| `results/tables/` | 结果表、统计表 |
| `docs/` | 数据来源、分析计划、运行记录、方法草稿 |

## 5 关键原则

### 5.1 原始数据不改

`data/raw/` 只读保存，下载什么样就尽量保留什么样。

### 5.2 中间对象放 `data/processed/`

例如：

```text
data/processed/01_read_qc/seurat_raw_qc.rds
data/processed/02_filter_qc/seurat_filtered.rds
data/processed/03_clustering/seurat_clustered.rds
data/processed/04_major_annotation/seurat_annotated.rds
```

### 5.3 图表拆开，再按步骤管理

推荐：

```text
results/
├── figs/
│   └── 01_read_qc/
└── tables/
    └── 01_read_qc/
```

### 5.4 `metadata/` 只放正式输入

适合放：

- 样本分组表
- donor 信息
- GEO series matrix
- 字段说明

不适合放：

- metadata 检查表
- 样本数统计表
- 临时预览文件

这些放在：

```text
results/tables/00_metadata_check/
```

## 6 路径写法

推荐：

```r
project <- path.expand("~/Projects/ProjectName")

raw_dir <- file.path(project, "data/raw")
meta_file <- file.path(project, "metadata/Metadata.Ind.csv")
fig_dir <- file.path(project, "results/figs/01_read_qc")
table_dir <- file.path(project, "results/tables/01_read_qc")
```

## 7 脚本命名

```text

```

例如：

```text

02_filter_qc.R

04_annotate_major_celltypes.R
05_compare_cell_fractions.R
06_run_deg_by_celltype.R
```

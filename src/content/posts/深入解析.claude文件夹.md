.claude 这个目录，是 Claude 的控制中心，存储规则以及记忆。

## 1 .claude目录有两种

一种是在每个项目内部，一种是在~/.claude 目录
![file 20260614225338931|734x398](https://pub-b6575bc5365d47eea85c3b697ba6ad51.r2.dev/2026/06/14/file-20260614225338931.png)

项目级别的.claude 放在具体的项目目录下，提交到 Git，协作者所有人都能获得相同的配置，包括设定的规则、skills、权限甚至记忆等等。

## 2 CLAUDE.md

这是整个 .claude 框架中最重要的文件。当启动一个 Claude Code 会话时，首先读取的就是CLAUDE.md。它会将内容直接加载到 system prompt 中，并在后续整个对话过程中保持在上下文中。

在项目根目录放一个 `CLAUDE.md` 是最常规的设置。也可以在 `~/.claude/CLAUDE.md` 放一个全局的，用于夸所有项目生效的偏好设置。Claude 会读取所有这些文件并合并。

## 4 rules /文件夹

`.claude/rules` 下的每个.md 文件都会与 CLAUDE.md 一起自动加载，可以按关注点拆分指令，如下：
```plaintext
.claude/rules/
├── code-style.md
├── testing.md
├── api-conventions.md
└── security.md
```
每个文件保持专注且易于更新。负责 API 规范的团队成员编辑 `api-conventions.md`，负责测试标准的人编辑 `testing.md`，互不干扰。

可以设置**路径作用域规则**，在规则文件中添加 YAML frontmatter，在 Claude 处理匹配文件路径时才会激活：
```markdown
---
paths:
  - "src/api/**/*.ts"
  - "src/handlers/**/*.ts"
---
# API 设计规则

- 所有 handler 返回 { data, error } 格式
- 使用 zod 做请求体校验
- 永远不要向客户端暴露内部错误详情
```
## 5 commands/文件夹

Claude Code 内置了一些 slash 命令，比如 `/help` 和 `/compact`。`commands/` 文件夹让你可以添加自己的命令。

你放入 `.claude/commands/` 的每个 Markdown 文件都会变成一个 slash 命令。

名为 `review.md` 的文件会创建 `/project:review`，名为 `fix-issue.md` 的文件会创建 `/project:fix-issue`。文件名就是命令名。

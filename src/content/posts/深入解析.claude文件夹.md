.claude 这个目录，是 Claude 的控制中心，存储规则以及记忆。

## 1 .claude目录有两种

一种是在每个项目内部，一种是在~/.claude 目录
![file 20260614225338931|734x398](https://pub-b6575bc5365d47eea85c3b697ba6ad51.r2.dev/2026/06/14/file-20260614225338931.png)

项目级别的.claude 放在具体的项目目录下，提交到 Git，协作者所有人都能获得相同的配置，包括设定的规则、skills、权限甚至记忆等等。

## 2 CLAUDE.md

这是整个 .claude 框架中最重要的文件。当启动一个 Claude Code 会话时，首先读取的就是CLAUDE.md。它会将内容直接加载到 system prompt 中，并在后续整个对话过程中保持在上下文中。

在项目根目录放一个 `CLAUDE.md` 是最常规的设置。也可以在 `~/.claude/CLAUDE.md` 放一个全局的，用于夸所有项目生效的偏好设置。Claude 会读取所有这些文件并合并。
